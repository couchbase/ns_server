You may filter for values and x-attributes in addition to key names. If the filter matches, the document will be replicated; otherwise the document won’t be replicated.

See the XDCR documentation for possible filter expressions (NOTE: regex pattern-matching and many N1QL statements are supported, but there are some unique aspects to the XDCR filtering language).

You may test if your filter expression matches a document by entering the document ID in the test field.

IMPORTANT: The filter expression does not apply to document delete operations or expirations; the filter expression will not stop any deletes or expirations from replicating. This is true even if the filter expression only applies to the document ID.

By default, filter expressions are not evaluated for documents that are deleted or expired. This means that all document deletions and expirations are replicated to the target regardless of whether the original document would have matched the filter or not.

If you are using a key-only filter expression, you can enable this option to have the filter expression applied to the deleted/expired document (also known as a tombstone document), so that only deletes and expirations that match the filter expression are replicated to the target. The binary documents option is used to specify whether binary documents should be replicated.
If the option is selected, binary documents are not replicated, regardless of whether a filter expression is applied.
If the option is not selected, the behavior depends on the conditions noted in the documentation.

If you select this option, the REST API filterBinary=true, and binary documents will not be replicated (regardless of whether or not you have a filter expression).

If you do not select this option, the REST API filterBinary=false (default), and binary documents may or may not be replicated, based on whether or not a filter expression exists:

  • If a filter expression does not exist, binary documents are replicated.
  • If a filter expression exists and contains key or xattrs only expression, binary documents are checked against the filter expression.
  • If a filter expression exists and contains references to the document body, binary documents are replicated.
