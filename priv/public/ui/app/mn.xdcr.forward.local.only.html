Only forward local mutations
  • In a full-mesh topology, this setting can reduce network bandwidth usage and data transfer costs.
  • Even in a full-mesh topology, this setting reduces redundancy by limiting each mutation to a single replication path to a given destination.
  • Both the source and target buckets must have Cross Cluster Versioning enabled.
  • All nodes in the replication topology must be running with cluster compatibility version 8.0.2 or later.
  • Use this setting only in a replication topology where every bucket is in bi-directional replication with every other bucket (otherwise the locally mutated data will not replicate to every bucket/cluster in the replication topology)
  • This setting does not support Mobile buckets in the replication topology.
Use this only in full-mesh active-active topologies. If enabled in other topologies, it can cause data loss.
